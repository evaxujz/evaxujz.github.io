递增的三元子序列

Posted on 2018-10-13 | In 算法

1.题目

给定一个未排序的数组, 判断这个数组中是否存在长度为3的递增子序列.

数学表达式如下:

如果存在这样的 i, j, k, 且满足 0 ≤ i < j < k ≤ n-1,
使得 arr[i] < arr[j] < arr[k], 返回 true; 否则返回 false.
说明: 要求算法的时间复杂度为 O(n), 空间复杂度为 O(1).

2.思路

在O(n)的时间复杂度要求下, 只能通过额外的变量存储信息来通过空间换取时间

可以通过求取n-1元数组的相关信息来辅助求取n元数组是否存在三元递增子序列,
这样每增加一个元素, 只需增加常数量级的操作,便可以获取新的结果, 在时间复杂度上便是O(n)
获取的n-1元数组的相关信息是:

1. n-1元数组是否存在三元递增子序列
2. n-1元数组的最小二元递增子序列

在知道这两种额外信息后便能够在O(1)的时间复杂度内确认n元数组是否存在三元递增子序列.
若n-1元数组已存在三元递增子序列则n元数组直接返回true, 若n-1元数组不存在三元递增子序列,
则通过确认n-1元数组的最小二元递增子序列与第n个元素是否为三元递增子序列, 是则返回true,
否则更新n元数组的最小二元递增子序列, 迭代到最后一个元素便能够确认目标数组是否存在三元递增子序列.

4.更简洁的思路

用两个变量MIN1, MIN2来表示当前元素Pn前的P1 - Pn-1元素的最小递增子序列.

MIN1 = Integer.MAX_VALUE, MIN2 = Integer.MAX_VALUE.

用最大值来作为初始值, 若没有被更新则说明不存在二元递增子序列或二元递增子序列的第二个元素为
Integer的最大值, 这两种结果都不存在三元递增子序列.

遍历数组元素, 若元素大于MIN2, 说明此时存在三元递增子序列, 否则比较元素与MIN1的大小, 若比MIN1
小则更新MIN1, 由于总是先更新的MIN1, 所以更够保证MIN1 < MIN2, 所以若MIN2小于Integer.MAX_VALUE则
说明已存在二元递增子序列, 此时再更新MIN1, MIN1作为暂存值并不会影响之前的二元递增子序列, 若MIN2再次更新,
则此时MIN1, MIN2就作为新的二元递增子序列
