JUC-ConcurrentHashMap

Posted on 2018-09-24 | In Java

1.ConcurrentHashMap产生的背景

HashMap虽然能够满足大部分场景的使用, 但是由于其本身是线程不安全的, 所以在多线程环境下使用HashMap可能会产生各种各样非预期的结果. 虽然Java也提供了线程安全的HashTable, 但是HashTable在执行同步操作时需要锁住整个Map, 所以效率十分低下. 因此需要一种既能保证线程安全又能显著提升效率的数据结构.

2.ConcurrentHashMap的原理

1.8之前ConcurrentHashMap是使用锁分段技术来实现的, 内部维护一个Segment数组, 该数组的每个元素都是一个HashTable, 通过将锁的粒度从整个Map细化到每个Segment一个锁, 在保证线程安全的前提下提高并发操作的效率.
1.8之后ConcurrentHashMap则使用Synchronized关键字与CAS操作来保证多线程下操作的安全性.
